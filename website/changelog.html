How v2 came together.

The full project shipped in eight phases — recon, then seven build phases. Each phase merged behind a git tag and was verified from a clean clone before the next began. The complete record lives in CHANGELOG.md.

Phase 7 — Release engineering tagged

Tag v2-phase-7-release · commit 775cd54

  • Quickstart, architecture, plugin-author, migration docs (docs/).
  • Packaging — Dockerfile, Homebrew formula, AUR PKGBUILD, Debian control.
  • CI matrix: Linux gcc / clang, macOS clang, Windows MSVC, ASan + UBSan + TSan presets.
  • Release-binary job: tagged builds publish tarballs as artifacts.
  • ADR 0010 — packaging + CI release matrix.

Phase 6 — CLI + plugin + migrator

Tag v2-phase-6-cli · commit 74452f1 · 271/271 tests

  • Hand-rolled argparse (--flag=val, --flag val, bundled bools, --).
  • nift binary with subcommands: init, build, serve, migrate, clean, version, help.
  • C-ABI plugin loader via dlopen/LoadLibrary; PluginRegistry validates ABI version 1 on load.
  • v1 → v2 migrator translates nsm.config + tracked.json; non-destructive.
  • End-to-end verified: nift init + nift build renders public/index.html.
  • ADR 0009 — CLI design, plugin C ABI, v1 migrator.

Phase 5 — Dev server + asset pipeline

Tag v2-phase-5-server · commit da1f83e · 247 tests

  • cpp-httplib HTTP/1.1 server, MIME map, ephemeral-port support.
  • Live reload via polling /__nift/livereload token + 500 ms client poll. No WebSockets.
  • Recursive polling FileWatcher: Added/Modified/Removed, configurable ignores.
  • Asset helpers: image format sniff (PNG/JPEG/WEBP/GIF/SVG/AVIF), CSS minify, JS minify (literal-aware).
  • ADR 0008 — cpp-httplib + polling watcher.

Phase 4 — Build pipeline

Tag v2-phase-4-build · commit d264a19 · 213 tests

  • WorkStealingPool — N-worker thread pool, LIFO local + FIFO random-victim steal, exception-safe via futures.
  • MmapReader via mio; heap fallback for 0-byte / odd FS edge cases.
  • xsimd byte scanner (AVX2 / SSE4.2 / NEON), scalar tail. Measured 2.4× scalar on 1 MiB sparse markers.
  • Pipeline — discover → hash → cache check → build → write, returns BuildReport.
  • ADR 0006 (work-stealing) and ADR 0007 (mmap + xsimd).

Phase 3 — Runtime + project

Tag v2-phase-3-runtime · commit 20791a6 · 179 tests

  • LuaRuntime — sol2-based Lua 5.4 with PIMPL; sandboxed by default (base/string/table/math); enable_unsafe() opt-in.
  • Print capture into __nift_capture returned from exec().
  • Recursive-descent expression evaluator: arithmetic, comparison, logic, string concat, variables, parens.
  • ProjectConfig with JSON round-trip; BLAKE3 hashing utilities (hash_content, hash_file, order-independent hash_combined).
  • BuildCache — atomic save (tmp + rename), is-dirty diffing, JSON v1 schema.
  • ADR 0004 (Lua via sol2) and ADR 0005 (BLAKE3 + JSON cache).

Phase 2 — Parser rewrite

Tag v2-phase-2-parser · 124 tests

  • Context-aware lexer — only @ / $ are special in text mode; \@ / \$ escapes.
  • Token set: Text, Directive, VarSimple/Bracket/Expr, parens, comma, comments, EOF.
  • Recursive-descent parser handles @name{opts}(params) and block directives (@if / @for / @while).
  • Visitor evaluator covering @if / @for / @while / @def / @set / @write / @echo / @input / @content / @exit.
  • Multi-token param extraction so $var survives token boundaries inside parens.
  • ADR 0003 — context-aware lexer choice.

Phase 1 — Foundation

Tag v2-phase-1-foundation · 62 tests

  • CMake build system with presets: debug, release, asan, ubsan, tsan, ci.
  • vcpkg manifest pinning fmt, spdlog, Catch2; baseline locked.
  • core/ crate — modern rewrites of Path, FileSystem, StrFns, Quoted, DateTime, SystemInfo.
  • Expected<T,E> polyfill: std::expected when available, minimal fallback otherwise.
  • Tooling: .clang-format, .clang-tidy, .editorconfig, Justfile, GitHub Actions CI.
  • ADRs 0001 (CMake), 0002 (vcpkg), 0003 (C++20/23 baseline).

Phase 0 — Recon

Tag v2-phase-0-recon

  • Generated docs/recon/ reports: toolchain, LOC, deps, hotpaths, build system, risks.
  • ADR 0000 — how Architecture Decision Records are written and reviewed.
  • Findings: 54 core source files, ~33 K LOC, 845 KB. Parser.cpp = 446 KB / 16,949 LOC / single function read_and_process_fn spans 12,562 lines.
  • Zero cyclic includes at the core layer.
  • No CMake, no CI, no tests in v1 — all built from scratch in Phase 1.

Versioning policy

Releases follow Semantic Versioning. The C ABI version (NIFT_PLUGIN_ABI_VERSION) is a separate integer that bumps only on breaking ABI changes — minor and patch versions of Nift never invalidate compiled plugins.
