Documentation

Build static sites with C++ speed.

Nift v2 takes a project tree of content, layouts, and partials and renders it to a static output/ directory. This page covers the everyday workflow.

Quickstart

One command scaffolds a new site, then a second command builds it:

nift init my-site
cd my-site
nift build       # output/ ready to deploy
nift serve       # http://127.0.0.1:8080 with file-watch reload

Project layout

Every Nift project shares the same skeleton:

my-site/
├── nift.config         # site-wide settings
├── content/            # one file per page
├── layouts/            # outer page templates
├── partials/           # reusable fragments
├── public/             # static assets, copied as-is
├── plugins/            # optional Lua + native plugins
└── output/             # generated, never edit by hand

The build is one direction: content + layouts + partials → output/. You never edit output/; you regenerate it.

Configuration

nift.config is a flat key = value file. Here is the full set of options:

KeyDefaultPurpose
name—Project identifier; embedded in JSON cache

content_dircontentSource pages

partials_dirpartialsReusable fragments
public_dirpublicStatic assets, copied verbatim
cache_dir.niftBLAKE3 cache index location

Directive syntax

Templates are plain HTML/Markdown with two special markers: @ for directives and $ for variable expansion. Anything that is not a marker is passed through verbatim, byte for byte. 

@layout(default)             <!-- choose outer layout -->
@set(title, Hello)            <!-- bind a variable -->
@partial(footer)              <!-- inline a partial -->
@if(condition)…@endif         <!-- conditional -->
@for(i; 0; 5)…@endfor         <!-- counted loop -->
@while(cond)…@endwhile        <!-- conditional loop -->
@def(name)…@enddef            <!-- define a callable block -->
@write(path, contents)        <!-- write extra output -->
@input(rel/path.html)         <!-- splice another file -->
@content                      <!-- expansion point inside layouts -->

Variables

Bind with @set(name, value); reference with $name or ${expression}. 

@set(title, Hello world)
@set(year, 2026)
<h1>$title</h1>
<footer>© $year</footer>

Layouts and partials

Layouts wrap pages. Partials are fragments. Both are looked up by name in their respective directories. 

<!-- layouts/default.html -->
<!doctype html>
<html>
  <head><title>$title</title></head>
  <body>
    @content
    @partial(footer)
  </body>
</html>

<!-- partials/footer.html -->
<footer>© $year</footer>

Control flow

Both numeric loops and boolean conditions are supported:

@for(i; 1; 5)
  <li>Item $i</li>
@endfor

@if(featured)
  <span class="badge">Featured</span>
@endif

@for(name; start; end) is half-open: it runs from start to end - 1. The separator is a semicolon — commas inside expressions never interfere with argument parsing.

nift build

nift build walks content/, parses each file into an AST, evaluates it against the configured layout, and writes the result to output/. The build is idempotent — running it twice on unchanged inputs produces byte-identical output.

The pipeline:

  1. Mmap each input.
  2. SIMD-scan to locate directive markers.
  3. Lex + parse into an AST.
  4. Hash inputs with BLAKE3; consult the cache.
  5. Evaluate & write — work-stealing across CPU cores.

Incremental cache

The cache lives in .nift/cache.json. Each entry maps a content path to its BLAKE3 input hash and the hashes of every layout and partial it consumed. Touching a layout invalidates only the pages that used it. 

# Force a clean rebuild
rm -rf .nift output
nift build

nift serve

Bundled dev server. Watches for file changes, rebuilds the affected pages, and serves output/ over HTTP. 

nift serve --host 127.0.0.1 --port 8080

The server is a development server. It is not intended for production — deploy output/ behind nginx, Cloudflare, GitHub Pages, or any static host.

Lua plugins

Drop a .lua file in plugins/ and Nift will load it in a sandboxed sol2 state. The sandbox strips os, io, package, debug, require, dofile, and loadfile, then installs a memory cap and an instruction-budget hook. Plugins register directives via the nift table. 

-- plugins/uppercase.lua
nift.directive("upper", function(args)
  return string.upper(args[1] or "")
end)

Now @upper(hello) renders as HELLO.

Native plugins

For work that needs real CPU — image transcoding, custom hashing — ship a shared library that exposes the C ABI in nift/plugin/abi.h. Drop the resulting .so / .dylib / .dll in plugins/; Nift loads it via dlopen / LoadLibrary at startup.

See the Plugins page for the full ABI.

Migrate from v1

One command rewrites a legacy nsm.config to the v2 schema, preserving the original as .backup:

nift migrate ./old-project

The migrator never deletes files; it copies. If a key in the v1 file has no v2 equivalent, the migrator emits a warning naming the key.
