Frequently asked

Questions worth answering up front.

Honest answers to the questions that keep coming up. If yours is missing, open an issue and we will add it here.

Why a C++ static site generator in 2026?

Two reasons. First, the original Nift was a 12k-LOC C++ monolith and v2 is the rewrite of that codebase — switching languages mid-rewrite would have meant abandoning a decade of evaluation logic. Second, static site generation is a real compiler problem: a parser, a typed AST, an evaluator, and a build cache. Native binaries make the incremental loop fast enough that you stop noticing it.

If you want a Node-based generator, use one. If you want a binary you drop into CI and forget about, this is for you.

Do I need to know C++ to use it?

No. The CLI is the only surface you touch. Templates use a small directive language documented in Docs › Directive syntax. Plugins are written in Lua. C++ is only required if you choose to ship a native plugin.

How does the Lua sandbox work?

Plugins run in a sol2-managed Lua state with the dangerous standard libraries stripped: os, io, package, debug, require, dofile, loadfile. A custom allocator caps plugin memory; an instruction-budget hook bounds CPU. A plugin cannot read /etc/passwd, cannot call os.execute, and cannot loop forever.

The sandbox has its own Catch2 test suite — see tests/qa/security/ — which fails CI if any of those guarantees regress.

How fast is it?

The SIMD scanner clocks at 0.37 ns/byte vs 0.89 ns/byte scalar on a 1 MiB sparse-marker buffer (AVX2, measured by the bundled bench_simd_scanner). A cold incremental build of a 200-page project finishes in roughly 95 ms; a warm incremental build where one file changed finishes in roughly 6 ms.

Numbers are from commit d264a19 and re-measured every CI run.

Is the dev server safe to expose to the internet?

No. nift serve is a development server. It binds 127.0.0.1 by default, refuses path traversal and oversized URLs, and rejects write methods, but it has no TLS, no authentication, and no rate limiting. Deploy output/ behind nginx, Caddy, Cloudflare Pages, GitHub Pages, or any static host you already trust.

What happens if I press Ctrl-C mid-build?

The build aborts cleanly: in-flight workers stop after their current page, partial output files are deleted (writes go through a temp file before rename(2)), and lock files are released. A subsequent nift build resumes from cache and produces byte-identical output to a clean build. The signal-recovery suite asserts this on every CI run.

Can I use it with Markdown?

Yes — drop .md files into content/. Nift renders them through a CommonMark pass before evaluating directives, so you can mix Markdown body text with @layout, @if, and variable expansion freely.

Does it support TypeScript / SCSS / asset bundling?

Out of the box, no. Nift v2 is a static site generator, not a bundler. Run your asset pipeline of choice (esbuild, Vite, Tailwind CLI, sass) before nift build and let it drop bundled output into public/. Nift copies public/ to output/ verbatim.

How do I migrate a v1 project?

One command:

nift migrate ./old-project

The migrator rewrites nsm.config as a v2 nift.config, preserves the original as nsm.config.backup, and warns about any v1 keys without a v2 equivalent. Templates and partials usually work unchanged. See Docs › Migrate from v1.

Why a .nift directory? Can I delete it?

.nift/ stores the BLAKE3 cache index that powers incremental builds. Delete it any time — the next build will rebuild everything and re-populate the cache. Add it to .gitignore (it already is in nift init projects).

Is there a hosted preview / cloud product?

No. Nift is a CLI binary you run locally and in your CI. There is no telemetry, no account system, and no managed offering. The site you ship is whatever output/ contains after nift build.
