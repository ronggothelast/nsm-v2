Command-line reference

The nift CLI.

Nift exposes one binary, nift, with a handful of subcommands. All commands return exit 0 on success and a non-zero code on failure; --help works on every level.

Synopsis

nift <command> [options]

commands:
  build      Render content/ to output/
  serve      Run the dev server with file watching
  init       Scaffold a new Nift project in the current directory
  migrate    Rewrite a v1 nsm.config as a v2 nift.config
  version    Print version and exit

nift build

nift build [--source DIR] [--out DIR] [--no-cache] [--jobs N]
FlagDefaultEffect
--sourcecwdProject root containing nift.config
--outoutputOverride output_dir from nift.config
--no-cacheoffIgnore .nift/cache.json; rebuild every page
--jobs Ncpu countParallelism for the work-stealing pool
--quietoffSuppress per-file logs; still prints summary
--verboseoffLog every cache decision and worker steal

nift serve

nift serve [--host HOST] [--port PORT] [--watch-interval MS]
FlagDefaultEffect
--host127.0.0.1Address to bind. Use 0.0.0.0 in containers.
--port8080TCP port
--watch-interval250File-watch poll interval, milliseconds
--no-watchoffServe output/ as-is; do not rebuild

nift init

nift init <name> [--template basic]

Creates the directory if missing, drops in nift.config, a default layout, an index page, and a .gitignore.

nift migrate

nift migrate <path-to-v1-project>

Reads the v1 nsm.config, writes a v2 nift.config alongside it, and renames the original to nsm.config.backup. The migrator is idempotent: running it twice does nothing on the second pass.

nift version

nift version

Exit codes

CodeMeaning
0Success
1Generic failure (parse error, IO error, plugin failure)
2CLI usage error (unknown flag, missing argument)
3Configuration error (missing nift.config, invalid key)
4Sandbox violation (Lua plugin attempted disallowed call)
5IO permission denied
130Interrupted by SIGINT

VarEffect
