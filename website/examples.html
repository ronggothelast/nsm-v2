Examples

Reference projects to read.

Each example is a full Nift project you can clone, build, and tear apart. They live in examples/ in the main repo and are exercised by CI on every push.

01 / Minimal blog

Markdown posts, RSS feed, dated archive

Eight files, one layout, one partial. Demonstrates Markdown content, per-page front matter, and a generated feed.xml via @write.

examples/blog

02 / Documentation site

Sidebar navigation, search index, code blocks

Mirrors the structure of this very docs site. Sidebar entries are generated from a JSON index; pages are written in Markdown with @partial(toc) for the on-page table of contents.

examples/docs

03 / Marketing one-pager

Single-page split-screen with asymmetric grid

Pure HTML in content/index.html, a layout, and a public/ directory of optimised images. Builds in under 4 ms. Works as a deployment smoke test for new toolchains.

examples/marketing

04 / Portfolio

Project gallery driven by a JSON manifest

Demonstrates a Lua plugin that reads a JSON file at build time and loops over it with @for. Sandbox-safe — the plugin uses the JSON helper exposed by the runtime, never io.open.

examples/portfolio

05 / Multi-language site

One project, two locales, two output trees

Uses the locales/ convention to render output/en/ and output/id/ from a single content tree. Showcases @input for shared layouts and @set for locale strings.

examples/i18n

06 / Native plugin host

C ABI plugin: an image-resize directive

A small C++ shared library that registers an @image directive. Demonstrates the C ABI, dynamic loading, and how to ship a plugin alongside the project that uses it.

examples/native-plugin

Run any example

git clone --recursive https://github.com/ronggothelast/nsm-v2.git
cd nsm-v2/examples/blog

nift build
nift serve     # http://127.0.0.1:8080

The examples assume nift is on your PATH. If you built from source, point at the binary directly: ../../build/release/apps/nift/nift build.
