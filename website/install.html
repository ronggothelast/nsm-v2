Installation

Install Nift v2

Nift v2 ships as a single static binary. No runtime dependencies, no package manager required. Pick your platform below.

Pre-built binary (recommended)

# Download latest release
curl -LO https://github.com/ronggothelast/nsm-v2/releases/latest/download/nift-linux-x86_64.tar.gz

# Extract and install
tar xzf nift-linux-x86_64.tar.gz
sudo mv nift /usr/local/bin/

# Verify
nift --version

Build from source

Requires CMake ≥ 3.25, a C++20 compiler (GCC 11+, Clang 14+, MSVC 2022+), and Git.

# Clone with submodules (vcpkg)
git clone --recurse-submodules https://github.com/ronggothelast/nsm-v2.git
cd nsm-v2

# Configure (downloads deps via vcpkg automatically)
cmake --preset release

# Build
cmake --build --preset release

# Install
sudo cp build/release/apps/nift/nift /usr/local/bin/  # Linux/macOS
# or: copy build\Release\nift.exe to your PATH       # Windows

# Verify
nift --version

Quick start

Create your first site in 60 seconds

# Create a new project
nift init my-blog
cd my-blog

# Build it
nift build
# Built 1 · cached 0 · failed 0 in 0 ms

# Start the dev server
nift serve
# Serving public/ at http://127.0.0.1:8080
# Watching content/ for changes

This scaffolds a project with nift.json, content/, and templates/. Edit files in content/ — the browser reloads automatically.

Troubleshooting

Common issues

"command not found: nift" after install

The binary is not on your $PATH. Either move it to a directory that is on your PATH, or add its location:

export PATH="$PATH:/path/to/nift/dir"

Add this to your ~/.bashrc or ~/.zshrc to make it permanent.

vcpkg submodule is empty after clone

You cloned without --recurse-submodules. Fix:

git submodule update --init --recursive

CMake version too old

Nift v2 requires CMake ≥ 3.25. Install a newer version:

pip3 install cmake (gets latest, works on all platforms)

Windows: "VCRUNTIME140.dll not found"

Install the Visual C++ Redistributable (x64). The Nift binary is built with MSVC 2022.
